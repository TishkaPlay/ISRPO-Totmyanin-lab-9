# Лабораторная работа №9: Адаптивная верстка с медиа-запросами

## Цель работы
Изучить принципы адаптивной верстки и медиа-запросов. Создать адаптивную страницу портфолио, которая корректно отображается на всех устройствах.

## Теоретическая часть

### Адаптивная верстка
Подход к созданию веб-страниц, при котором дизайн и контент автоматически подстраиваются под размер экрана устройства (компьютер, планшет, смартфон).

### Медиа-запросы (Media Queries)
CSS-техника, позволяющая применять разные стили в зависимости от характеристик устройства:
- Ширина экрана
- Ориентация (портретная/ландшафтная)
- Разрешение

### Синтаксис
```css
@media (условие) {
  /* стили для этого условия */
}
```

### Подходы к адаптивности
- **Mobile First** — начинаем с мобильных стилей, затем расширяем для больших экранов (рекомендуется)
- **Desktop First** — начинаем с десктопных стилей, затем адаптируем для маленьких экранов

### Стандартные контрольные точки (breakpoints)
- Мобильные устройства: до 768px
- Планшеты: 768px – 991px
- Десктопы: 992px – 1199px
- Большие десктопы: от 1200px

## Практическая часть

### Созданные файлы
- `index.html` — основная страница портфолио
- `styles.css` — базовые стили
- `responsive.html` + `responsive.css` — пример адаптивного макета с использованием Flexbox и Grid
- `nav-practice.html` + `nav-practice.css` — адаптивная навигация (Самостоятельное задание №1)
- `flexbox-practice.html` + `flexbox-practice.css` — Flexbox-макет (Самостоятельное задание №2)

### Особенности реализации
1. Использован подход **Mobile First**
2. Применены медиа-запросы для адаптации под разные устройства
3. Для макетов использованы современные технологии:
   - Flexbox для одномерных раскладок (навигация, карточки)
   - CSS Grid для двумерных раскладок
4. Все изображения адаптивны (`max-width: 100%`)
5. Добавлены плавные переходы и эффекты при наведении

## Тестирование
Страницы протестированы в режиме адаптивного дизайна браузера на следующих разрешениях:

- Планшеты (768px)
